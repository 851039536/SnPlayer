## 修复并行加密/解密版本字节截断 bug 并启用多线程加密

### 根因
`crypto_service.dart` 中 `_writeBatchToOutput` 使用 `FileMode.write`（等同 `O_TRUNC`）打开输出文件。并行加密流程第 5 步先写入 64 字节文件头（含版本号 0x02），关闭文件；随后 `_writeBatchToOutput` 重新用 `FileMode.write` 打开 → **截断！文件头清零，版本字节变 0x00**。此外 `parallelDecryptMaxConcurrency=2`，当 isolateCount>2（256MB+ 文件）时 `_writeBatchToOutput` 被多次调用，每次截断导致前批 chunk 数据也丢失（解密同样受影响，只是之前未暴露）。

### 修复（`lib/services/crypto_service.dart`，3 处改动）
1. **`_writeBatchToOutput`** — 签名改为接收 `RandomAccessFile outputRaf`（已打开句柄），移除内部 `openSync`/`closeSync`，变为纯写入逻辑，生命周期由调用方管理
2. **`_runParallelEncrypt`** — 第 5 步写 header + 预分配后**保持句柄打开**，各批次复用同一句柄调 `_writeBatchToOutput(outputRaf, ...)`，`try/finally` 保证异常时关闭
3. **`_runParallelDecrypt`** — 同上，第 4 步预分配后保持句柄打开，`try/finally` 保护

### `encryptFile` 自动选路
`encryptFile` 已是自动选择路径：>= `parallelDecryptMinFileSize`（64MB）走 `encryptFileParallel`，否则串行。调用方 `video_list_provider.dart` 零改动，透明升级。

### 文档更新
`CODEBUDDY.md` 移除"并行加密代码保留但不应被启用"的警告，更新为已修复并自动启用说明。

## 加密功能代码审查修复与加速

### 高优逻辑漏洞修复
1. **并行失败资源清理**（`_runParallelEncrypt`/`_runParallelDecrypt`）：try/finally 改为 try/catch/finally，catch 块清理残留临时 chunk 文件 + 删除损坏的输出文件 + rethrow。原先异常时临时文件和半成品输出文件残留磁盘。
2. **并行进度回调单调化**：维护 `maxProgress` 变量，回调时取 `max(old, new)`。原先两 chunk 并发交错执行导致进度值来回跳（0.3→0.05→0.35→0.1）。

### 中优一致性修复
3. **`decryptFile` 自动选路**：与 `encryptFile` 一致，>= 64MB 走 `decryptFileParallel`，否则串行。
4. **PBKDF2 注释修正**：实际 `pbkdf2Iterations=10`，但注释写"100K"/"1万次"/"100,000 次"。统一修正为 10 次并说明原因（固定密码场景，安全性靠密码强度）。
5. **class doc 更新**：2-4 路改为 2-6 路，"解密"改为"加解密"。

### 加速优化
6. **去重复 stat**：`encryptFile` 已获取 `fileSize`，通过 `encryptFileParallel` → `_runParallelEncrypt` 透传，避免内部重复 `File(inputPath).length()`。
7. **移除多余 delay**：删除每批 chunk 完成后的 `Future.delayed(Duration.zero)`（Future.wait 和 _writeBatchToOutput 已有 await 让出点）。

### 代码去重
8. **合并 `_spawnEncryptChunkIsolate` + `_spawnChunkIsolate`** → 通用 `_spawnChunkIsolateGeneric(isEncrypt: bool, writeOffset: int?)`，减少约 60 行重复（`crypto_service.dart`）。
9. **合并 `_encryptChunkInIsolate` + `_decryptChunkInIsolate`** → 通用 `_processChunkInIsolate(isDecrypt: bool)`，减少约 80 行重复（`crypto_isolate.dart`）。Worker 派发同步更新。

## 播放解密代码审查修复

### 高优内存泄露修复
1. **`_initWithProxy` _controller 泄露**（`video_player_screen.dart`）：`_initWithProxy` 内部 `initialize()`/`play()` 抛异常时 `_controller` 未 dispose，导致 ExoPlayer 原生资源泄露。修复：内部加 try-catch dispose + rethrow，外层 `_initPlayer` 阶段 2 catch 补充 dispose。
2. **worker cancelled 全局竞态**（`streaming_decrypt_proxy.dart`）：全局 `bool cancelled` 被所有 decrypt_range 共享，并发 Range 请求（视频轨+音频轨）互相干扰——请求 B 到达时 `cancelled=false` 会"取消取消"请求 A，seek 时 `cancelled=true` 误取消所有请求。修复：改为 `int? cancelledRequestId` 匹配机制，每个请求分配唯一 ID，cancel 携带目标 ID，`isCancelled` 检查 `cancelledRequestId == requestId`。

### 中优修复
3. **`_playExternal` NO_PLAYER 分支泄露**（`video_list_screen.dart`）：openUrl 抛 NO_PLAYER 时直接 return，`_externalProxy` 未清理。修复：return 前 `await _externalProxy?.stop(); _externalProxy = null;`。
4. **`stop()` 冗余 send**（`streaming_decrypt_proxy.dart`）：`send('stop')` 后立即 `kill(immediate)`，消息不被处理。移除冗余 send。
5. **dispose unawaited**（两个 screen）：`_proxy.stop()` / `_externalProxy.stop()` 改为 `unawaited(...)` 明确 fire-and-forget 意图。

### 文件变更
- `lib/services/streaming_decrypt_proxy.dart` — worker requestId 机制 + stop 冗余清理
- `lib/screens/video_player_screen.dart` — _controller 泄露修复 + unawaited
- `lib/screens/video_list_screen.dart` — NO_PLAYER 清理 + unawaited
