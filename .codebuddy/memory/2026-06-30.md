## SnPlayer 加解密性能优化

优化了 AES-256-CTR 加解密的 3 个性能瓶颈，解决播放和导出解密卡顿问题：

### 修改文件
1. **`lib/services/crypto_service.dart`** — 3 处改动：
   - `_processCtrBlock()`：逐字节 `cipher.returnByte()` 循环 → `cipher.processBytes()` 批量处理（吞吐量提升 100-1000 倍）
   - `deriveKey()`：新增 PBKDF2 密钥缓存 `_keyCache`，以 salt base64 为 key，避免相同文件重复 100K 迭代
   - `encryptFile()`/`decryptFile()`：改为 `Isolate.spawn(cryptoWorker)` 后台执行，主线程零阻塞
   - 新增 `_runInIsolate()` 统一封装 Isolate 通信
   - 删除未使用的 `xorBytes` 函数

2. **`lib/services/crypto_isolate.dart`** — 新建文件（~210 行）：
   - `cryptoWorker()` 顶层函数作为 Isolate 入口
   - 通过 `SendPort`/`ReceivePort` 双向通信：主线程发命令，worker 回传进度
   - 独立的密钥缓存 `_isoKeyCache`（跨 Isolate 隔离）
   - 双缓冲 I/O 流水线 + 批量 CTR 处理

### 技术要点
- PointyCastle `StreamCipher.processBytes()` 批量 API 替代逐字节循环
- `Isolate.spawn()` + `ReceivePort` 实现后台加解密 + 实时进度回传
- `encryptBytes()`/`decryptBytes()`（缩略图）保持主线程，数据量小 Isolate 开销不划算
- 公共 API 签名零变更，调用方（video_player_screen、video_list_provider、thumbnail_service）无需修改
- lint 0 error

## 播放失败 Bug 修复（Isolate 提前被杀）

**现象**：播放加密视频时报 `UnrecognizedInputFormatException: None of the available extractors could read the stream`

**根因**：`crypto_isolate.dart` 中 3 处 `async` 缺失导致解密未完成就发出 `done` 信号，主线程随即杀死 Isolate，输出文件残缺。

修复（3 处改动）：
1. `receivePort.listen` 回调改为 `async`，对 `_encryptFileInIsolate`/`_decryptFileInIsolate` 加 `await`
2. `_encryptFileInIsolate`/`_decryptFileInIsolate` 改为 `Future<void> async`，`await _processFile()`
3. 解密时将 `raf.closeSync()` 提前到 `_processFile` 之前执行，避免双文件句柄冲突

## 导出文件潜在数据损坏修复（sublistView → sublist）

**问题**：`_processFile` 第 166 行使用 `Uint8List.sublistView(procBuf, 0, bytesRead)` 传给 `output.add()`。
`sublistView` 创建视图（不复制），而 `procBuf` 每轮循环被 `cipher.processBytes()` 覆盖。
若 `IOSink` 尚未 flush 前一帧数据，持有视图的底层缓冲会被下一帧改写，导致文件内容交错损坏。

**修复**：改为 `procBuf.sublist(0, bytesRead)`（深拷贝），与原同步版代码行为一致。
性能影响可忽略（每次多一次 512KB memcpy，远小于 I/O 开销）。

## 视频列表空UI修复

**现象**：视频卡片下半部分大面积空白。

**根因**：`SliverGridDelegateWithFixedCrossAxisCount` 的 `childAspectRatio: 0.75` 过小。
2 列每卡宽 ≈184px，按 0.75 算高 = 245px，但实际内容只需 ~180px，每个卡片底部空出 ~65px。

**修复**（2 处改动）：
1. `video_list_screen.dart:233` — `childAspectRatio: 0.75` → `1.0`
2. `video_card.dart:47` — info 区 padding `EdgeInsets.all(10)` → `EdgeInsets.fromLTRB(8, 8, 8, 6)`

## 视频列表下半部空 UI 修复

**问题**：网格 `childAspectRatio: 0.75` 导致每张卡片被强制拉伸到 245px 高，而实际内容只需要 ~170px，
每个卡片底部多出 ~75px 空白，多行叠加后下半屏幕大面积空白。

**修复 2 处**：
1. `video_list_screen.dart:233` — `childAspectRatio` 从 `0.75` 改为 `1.0`（匹配 16:9 缩略图 + 紧凑信息区的真实宽高比）
2. `video_card.dart:47` — 信息区 padding 从 `EdgeInsets.all(10)` 改为 `EdgeInsets.fromLTRB(8, 8, 8, 6)`（底部收紧 4px）
