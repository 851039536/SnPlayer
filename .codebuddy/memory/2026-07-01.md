## openFolder 修复（两次迭代）

### 第一次：file:// URI → SAF content URI
- 根因：Android 7+ 禁止通过 Intent 暴露 `file://` URI
- 修复：`Uri.fromFile()` → `DocumentsContract.buildTreeDocumentUri()` + `FLAG_GRANT_READ_URI_PERMISSION`
- 结果：仍然失败，报错 `UID does not have permission to content://...tree/...`

### 第二次：SAF 无权限 → ACTION_OPEN_DOCUMENT_TREE 三级回退
- 三级策略：file:// URI (Android 6) → SAF content URI (Android 7-9) → ACTION_OPEN_DOCUMENT_TREE + EXTRA_INITIAL_URI (Android 10+)
- 新增 `pendingFolderResult` 字段 + `onActivityResult()` 重写，处理 SAF 授权回调
- 授权后用 `takePersistableUriPermission()` 持久化，下次免授权
- 文件：`android/app/src/main/kotlin/.../MainActivity.kt`（完全重构 openFolder 逻辑）
